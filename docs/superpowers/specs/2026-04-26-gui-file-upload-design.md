# GUI 文件提交功能设计

## 概述

为 GUI 版本添加文件提交功能，复用 CLI 版本已实现的 FileManager 类，允许用户在对话中上传文件供 AI 分析。

## 需求

- 在输入框右侧添加文件选择按钮
- 在输入框上方显示已添加的文件列表
- 支持点击 × 删除单个文件
- 发送消息时自动携带文件附件
- 支持文本文件、图片、二进制文件

## UI 布局

```
┌───────────────────────────────────────────────┐
│  聊天内容区域                                  │
├───────────────────────────────────────────────┤
│  ┌─────────────────────────────────────┐      │  ← 文件列表区域（有文件时显示）
│  │ [file.txt ×] [image.png ×]          │      │
│  └─────────────────────────────────────┘      │
│  ┌────────────────────┬──────┬────────┐       │
│  │ 输入框             │ 📁   │ 发送   │       │  ← 文件按钮在发送按钮左侧
│  └────────────────────┴──────┴────────┘       │
└───────────────────────────────────────────────┘
```

## 新增组件

| 组件 | 类型 | 说明 |
|------|------|------|
| `m_fileManager` | `FileManager*` | 文件管理器（复用 CLI 版本） |
| `m_fileButton` | `QPushButton*` | 文件选择按钮（图标按钮） |
| `m_fileListArea` | `QWidget*` | 文件列表显示区域 |

## 数据流

```
用户点击文件按钮 → QFileDialog 选择文件 → FileManager.addFile()
                                                    ↓
发送消息时 ← FileManager.pendingFiles() ← 文件列表显示更新
                                                    ↓
                                    NetworkManager 构建多模态 JSON → API
```

## 交互流程

### 添加文件

1. 用户点击文件按钮
2. 弹出 `QFileDialog::getOpenFileNames`（支持多选）
3. 验证文件（存在、大小 ≤10MB）
4. 添加成功后，在文件列表区域显示文件标签
5. 标签格式：`[文件名 ×]`，点击 × 删除该文件

### 发送消息

1. 用户输入文字，点击发送
2. 如果有待发送文件，显示提示"发送消息时携带 X 个文件"
3. 构建带附件的 ChatMessage
4. 发送后自动清空文件列表

### 错误处理

- 文件不存在：弹出 `QMessageBox::warning`
- 文件过大（>10MB）：弹出 `QMessageBox::warning`
- 非文本/图片文件：显示为"二进制"类型标签

## 文件标签样式

每个文件标签包含：
- 文件名（简短显示，过长截断）
- 文件类型标识
- 删除按钮（×）

**颜色区分：**
- 文本文件：蓝色边框
- 图片文件：绿色边框
- 二进制文件：灰色边框

## 与 CLI 版本的差异

| 特性 | CLI | GUI |
|------|-----|-----|
| 文件选择方式 | `/file <路径>` 命令 | 点击按钮 + 文件对话框 |
| 文件列表显示 | `/listfiles` 命令 | 输入框上方可视化列表 |
| 清空文件 | `/clearfiles` 命令 | 发送后自动清空 + 点击 × 删除单个 |
| 文件类型复用 | FileManager | 同 FileManager |
| 消息格式 | NetworkManager 多模态 | 同 NetworkManager |
